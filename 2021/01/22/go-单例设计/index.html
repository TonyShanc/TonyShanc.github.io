go 单例设计

涉及到数据库/grpc连接的struct/class的设计，考虑单例与否尤为重要。目前我接触的项目里grpc连接通常是单例（因为http2支持多路复用，单链接可以支持并发访问），针对db的连接通常会用到xorm，xorm的engine也是单例，它会帮我们管理db连接池（应该）。进一步封装grpcClient和 xorm.Engine了的struct，实现了持久层方法的同时也得是个单例。

初始化一个grpc连接

//靠编译时初始化的全局变量实现的单例，同时实现并发安全+懒加载(调用时执行)

var (
    kratos = new(kratosGRPC)
) 

type kratosGRPC struct{
  Client kratosGrpcClient
  mu sync.Mutex
  ...
}
func GetClient(){
  return kratos.getClient()
}
func (k *kratosGRPC)getClient()*kratosGrpcClient{
  // client已经被初始化
  if k.Client != nil {
    return k.Client
  }else {
    // 竞争抢锁初始化client
    k.mu.Lock()
    defer k.mu.Unlock()
       //  抢到锁后需要再判断
       if k.Client != nil {
            return k.Client
       }
    // 建立连接
    k.Client = grpc.Dial(...)
    ...
    return k.client
  }

}

上面是一种实现方式，当然也可以进行一些小改动，比如实现并发安全用Once.Do代替sync.Mutex。这种建立方式比较主流

在某个特殊的需求下，可以这样生成单例

假设有个需求，经分析，要有多个job，每个job有固定的工作方式， job需要并发处理多个task, 就把这些task交给worker, worker根据task中的数据固定调用job的方法执行。

job涉及到数据库的数据处理，所以需要提供持久层服务，该服务最终会提供handle方法做固定的工作，由于我们只需要handle方法，所以可以隐藏持久层服务，具体方法是通过工厂方法只调用一次的性质初始化一次持久层服务，将持久层服务中的handle方法交给工厂方法的返回体job实例。由于handle被引用了，所以持久层的单例没有被GC回收，job.HandleFunc还是可以使用持久层提供的数据库连接的。这样，代码逻辑中只含有job、task、work。

做法如下：

type HandleFunc func ()
type job struct {
    Dsn string
    TableName string
    From int
    To int
    ConCurrentNum int
    HandleFunc HandleFunc
}
type RWHandler interface {
    Handle()
}
type userDataRWHandler struct {
    //这里含有众多处理数据库连接的handler
}
type adminDataRWHandler struct {
    //这里含有众多处理数据库连接的handler
}
func (handler userDataRWHandler) Handle(){
    fmt.Println("userDataRWHandler handling...")
}
func (handler adminDataRWHandler) Handle(){
    fmt.Println("adminDataRWHandler handling...")
}
func newUserDataHandler() RWHandler{
    return &userDataHandler{}
}
func newAdminDataHandler() RWHandler {
    return &adminDataHandler{}
}
type Config struct {
    HandlerName string
    Dsn string
    TableName string
    From int
    To int
    ConCurrentNum int
}
func newRWHandler(handlerName string)RWHandler {
    switch handlerName {
    case "userDataRWHandler":
        return newUserDataHandler()
    case "adminDataRWHandler":
        return newAdminDataHandler()
    default:
        return nil
    }
}
func NewJob(config Config)*job{
    handler := newRWHandler(config.HandlerName)
    return &job{
        Dsn: config.Dsn,
        TableName: config.TableName,
        From: config.From,
        To: config.To,
        ConCurrentNum: config.ConCurrentNum,
        HandleFunc: handler.Handle,
    }
}
