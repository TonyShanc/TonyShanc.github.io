找出数组中重复的数字

在一个长度为 n 的数组 nums 里的所有数字都在 0～n-1 的范围内。数组中某些数字是重复的，但不知道有几个数字重复了，也不知道每个数字重复了几次。请找出数组中任意一个重复的数字。

示例 1：

输入：
[2, 3, 1, 0, 2, 5, 3]
输出：2 或 3

限制：

2 <= n <= 100000

思路

很简单的一道题，想要访问重复数字，必须考虑存储之前访问过的数据来比较，同时考虑迭代过程中可进行前后比较，提供两种思路：

1: 访问过的数据存入map[int]bool, 存在为true不存在则为零值false。 时间复杂度O(n), 空间复杂度O(n)

2: 排序，前后比较判断，时间复杂度O(nlogn), 空间复杂度O(1)

复习一下各大排序算法：

代码

golang map+迭代 时间复杂度O(n), 空间复杂度O(n)

func findRepeatNumber(nums []int) int {
    m := make(map[int]bool)
    for _,n := range nums{
        if m[n] == false{
            m[n] = true
        }else{
            return n
        }
    }
    return -1
}

golang 快排+前后比较 时间复杂度O(nlogn),空间复杂度O(1)

func findRepeatNumber(nums []int) int {
    length := len(nums)
    sort(nums, 0,length-1)
    n := nums[0]
    for i:=1; i < length; i++{
        if nums[i]==n {
            return n
        }
        n = nums[i]
    }
    return 0
}
//快排
func sort(nums []int, start, end int){
    if start >= end {
        return 
    }
    p := partition(nums, start, end)
    t := solve(nums, start, end, p)
    sort(nums, start, t-1)
    sort(nums, t+1, end)
}
func solve(nums []int, start, end int, p int)int{
    i := start
    j := end - 1
    //坑点，要求i<=j
    for i <end && j>=i {
        if nums[i]<nums[p]{
            i+=1
        }else{
            //交换
            swap(nums, i, j)
            j-=1
        }
    }
    swap(nums, i, p)
    return i
}

结果超时，时间复杂度太高，说明不能用排序。

最后在书上看到了最优解 时间复杂度O(n), 空间复杂度O(1)

突破点在 数组 nums 里的所有数字都在 0～n-1 的范围内 这个条件中： 考虑把所有访问过的数字放到对应下标上，这种情况下若出现重复数字那会造成冲突，返回冲突值；若没有冲突，每次访问都会让一个数字归位，只会访问n次。

func findRepeatNumber(nums []int) int {
length := len(nums)
i := 0
for i < length {
    if nums[i]==i{
        i+=1
    }else{
        if nums[i] == nums[nums[i]]{
            return nums[i]
        }else{
           swap(nums, i, nums[i]) 
        }
    }
}
return -1
}
func swap(nums []int, i, j int){
    t := nums[j]
    nums[j] = nums[i]
    nums[i] = t
}
